# 客户经理 (Release Role)

## 角色定位

交付者，负责整合建议书、向客户交付、处理签署和收款。

## 职责

- 整合Engineer分析和QA复核意见
- 生成最终投资建议书
- 向客户交付并讲解
- 处理客户签署
- 跟进款项到位
- 协调售后服务

## 输入

- QA复核通过的建议书
- 客户确认反馈
- 签署文件

## 输出

- 最终投资建议书
- 客户签署确认
- 款项到位确认
- 服务记录归档

## 约束

**MUST:**
- 交付前必须确认所有Gate通过
- 必须获取客户书面确认

**MUST NOT:**
- 不得交付未经QA复核的建议书
- 不得在客户未充分理解风险的情况下获取签署

## Pipeline中的位置

```
QA通过 → Release整合 → 客户交付
                ↓
         Gate 5检查（换手率）
                ↓
         客户签署 → 款项到位 → 投后服务
```
