# 风险测评与解读 (Risk Assessment)

## Overview

基于KYC信息进行风险测评，计算风险等级，向客户解读结果。

**核心定位**：量化客户的实际风险承受能力，对应产品风险等级。

## When to Use

- KYC信息收集完成后
- 客户风险等级重新评估时
- 客户投资目标重大变更时

## 测评维度

| 维度 | 权重 | 说明 |
|------|------|------|
| 投资目标 | 30% | 保值/稳健/增值/高收益 |
| 风险承受能力 | 40% | 资产规模/收入稳定性/负债/家庭责任 |
| 投资经验 | 20% | 投资年限/产品熟悉度/风险认知 |
| 流动性需求 | 10% | 资金使用计划/封闭期接受度 |

## 评分规则

### 投资目标维度（30分）

| 选项 | 得分 |
|------|------|
| 本金保障，不接受亏损 | 0-10 |
| 稳健增值，少量波动可接受 | 11-20 |
| 均衡配置，适度风险 | 21-30 |
| 追求收益，可接受较大波动 | 31-40 |
| 高收益，可接受大幅波动 | 41-50 |

### 风险承受能力维度（40分）

| 因素 | 评估内容 | 得分 |
|------|---------|------|
| 资产规模 | 可投资资产/总资产比例 | 0-10 |
| 收入稳定性 | 职业稳定性/收入波动性 | 0-10 |
| 负债情况 | 负债率/偿债能力 | 0-10 |
| 家庭责任 | 抚养负担/养老负担 | 0-10 |

### 投资经验维度（20分）

| 投资年限 | 得分 | 产品熟悉度 | 额外得分 |
|---------|------|-----------|---------|
| <1年 | 0-3 | 无了解 | 0 |
| 1-3年 | 4-7 | 了解1-2类 | 1-3 |
| 3-5年 | 8-12 | 了解3-4类 | 4-6 |
| 5-10年 | 13-16 | 了解5类以上 | 7-10 |
| >10年 | 17-20 | 实操经验丰富 | +5 |

### 流动性需求维度（10分）

| 资金使用计划 | 得分 |
|------------|------|
| 1年内可能使用 | 0-2 |
| 1-3年内可能使用 | 3-5 |
| 3-5年内可能使用 | 6-8 |
| 5年以上不需要使用 | 9-10 |

## 风险等级划分

| 总分 | 风险等级 | 说明 | 可配置产品 |
|------|---------|------|-----------|
| 0-30 | C1（保守型） | 保本为主，低波动 | R1 |
| 31-60 | C2（稳健型） | 稳健增值，低风险 | R1, R2 |
| 61-80 | C3（平衡型） | 均衡配置，中等风险 | R1, R2, R3 |
| 81-100 | C4（成长型） | 追求收益，中高风险 | R1-R4 |
| 101-120 | C5（进取型） | 高风险高收益 | R1-R5 |

## 向客户解读话术

```
【C1保守型解读】
"根据您的测评结果，您的风险偏好属于保守型。这意味着您更看重资金安全，可以接受少量收益波动来换取资产保值。我建议以R1级产品为主，比如货币基金、短债基金等。"

【C3平衡型解读】
"您的风险偏好属于平衡型，可以接受适度的收益波动来追求资产增值。我建议以R2-R3级产品为主，配置一些混合基金、债券基金，适度参与权益市场。"

【C5进取型解读】
"您的风险偏好属于进取型，可以承受较大的本金波动来追求高收益。我建议以R4-R5级产品为主，配置股票型基金、指数基金等。"
```

## 合规要求

- 必须客户本人独立完成测评
- 测评结果需客户签字确认
- 存档记录

## Example

**测评示例：**

张先生，45岁，测评结果：
- 投资目标得分：28分（增值型）
- 风险承受能力：30分（资产充裕、收入稳定）
- 投资经验：16分（15年经验，多类产品）
- 流动性需求：6分（2年内有用款计划）

总分：80分 → **C3平衡型**

**向客户解读：**
> "张先生，根据您的测评结果，您的风险偏好属于平衡型。这意味着您可以接受适度的收益波动来追求资产增值，建议以R2-R3级产品为主，配置一些混合基金和债券基金，适度参与权益市场。"

**输出：**
```json
{
  "customer_id": "kyc_001",
  "assessment": {
    "total_score": 80,
    "risk_level": "C3",
    "suitable_products": ["R2", "R3"]
  }
}
```
